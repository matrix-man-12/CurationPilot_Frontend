CurationPilot — Project Overview

What is CurationPilot?

CurationPilot is an AI-powered operations automation platform. It converts manual browser-based workflows into executable, parameterized skills — Playwright scripts that can be triggered on demand through an API interface.

The Automation Pipeline

The platform bridges the gap between manual operations and automated workflows via a structured four-stage process:

Manual Workflow ➔ Playwright Script ➔ API Endpoint ➔ Frontend Interface
  1. Skill Creation: Human operators perform tasks manually in the browser while the execution engine records actions.
  2. Script Generation: The recorded workflow is compiled into a Playwright script with parameterized inputs.
  3. API Exposure: Each script is exposed as a REST API endpoint that accepts dynamic parameters and executes the automation runner.
  4. Frontend Execution: Business users configure and trigger automations through this chat-like user interface, providing inputs manually or in bulk (via CSV files).

Role of the Frontend

The frontend application serves as the user-facing command center. It is responsible for:

  • Listing Available Skills: Organized by category in a searchable dropdown selector.
  • Parameter Configuration: Generating dynamic forms (text, numeric, selects, booleans, arrays) based on the skill's parameter schema.
  • Bulk Execution Support: Ingesting and validating CSV files, rendering tabular previews, and converting rows into bulk executions.
  • Execution Control: Enabling operators to Pause, Resume, or Cancel tasks mid-execution.
  • Human-in-the-loop (HITL) Interactivity: Presenting confirmation modals during high-risk steps, enabling manual approval or rejection.
  • Real-Time Logs & Progress: Showing percentages and streaming logs as steps advance.
  • Session History: Saving and recovering past sessions from local storage.

Framework React 19 Functional components, hooks, Context API for state management.
Build Tool Vite 8 Ultra-fast development server and production bundler.
Styling Vanilla CSS Custom CSS Variables (Design Tokens) matching the Warm-Pastel design theme.
State & Persistence Context API & localStorage Local session mapping, logs history, and active run states persist across refreshes.
API Communication Fetch API & Short-Polling REST integration layer. Polling is utilized until WebSockets are fully integrated.
Real-Time Events WebSockets (Future Phase) Scheduled to replace polling to enable true low-latency log streams.
