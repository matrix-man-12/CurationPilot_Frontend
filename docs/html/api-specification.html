REST API Specification

Version: 1.1.0
Base URL: http://localhost:8080/api

Overview

This document defines the REST API contract between the CurationPilot frontend and backend. The frontend consumes these endpoints to list skills, submit single and bulk execution requests, toggle execution status (pause/resume/cancel), and handle human-in-the-loop verification (approve/reject).

Authentication

No authentication is required in the current version. Auth headers (JWT bearer tokens) will be added in a future security iteration.

Common Response Format

All REST API responses are wrapped in a standard JSON envelope:

{
  "success": true,
  "data": { ... },
  "error": null
}

Error responses (HTTP 4xx/5xx status codes) follow this envelope structure:

{
  "success": false,
  "data": null,
  "error": {
    "code": "SKILL_NOT_FOUND",
    "message": "Skill with ID 'skill_999' does not exist"
  }
}

Endpoints

1. List Skills

GET/api/skills

Returns all available skills grouped by category, with metadata counters.

Query Parameters:

Parameter Type Required Description
search string No Filter skills by name or description (case-insensitive partial match).

Response Data:

{
  "success": true,
  "data": {
    "categories": [
      {
        "id": "cat_data_extraction",
        "name": "Data Extraction",
        "skills": [
          {
            "id": "skill_001",
            "name": "Extract Invoice Data",
            "description": "Navigates to the invoicing portal and extracts line items for a given date range",
            "category": "Data Extraction",
            "parameterCount": 4,
            "lastExecuted": "2026-06-14T10:30:00Z",
            "executionCount": 47
          }
        ]
      }
    ],
    "totalSkills": 5
  }
}

3. Supported Parameter Types

The parameter type determines how the input element is rendered in the UI form:

Type Value UI Rendered Input Sent JSON Format
text Single line text input box "string value"
textarea Multi-line large textarea "multi\nline\nvalue"
number Numeric spinbox 123 or 1.5
url Validated text input URL field "https://example.com"
email Validated email text field "user@domain.com"
date Calendar date selector picker "2026-06-15" (ISO 8601 string)
select Single-select dropdown selection "option_value"
comma_separated Text field converting to Array ["val1", "val2"]
boolean Toggle slider switch true or false
password Masked password input text box "secret_token"
CSV Column Separator Parsing Rule

To prevent conflict with outer CSV delimiters, fields marked as comma_separated are split in CSV cells using **semicolons (;)** (e.g., V01;V02;V03). In manual form configuration inputs, standard **commas (,)** are used instead.

4. Submit Execution

POST/api/executions

Submits a Playwright automation execution request. Supports single and bulk runs.

Single Execution Payload:

{
  "skillId": "skill_001",
  "parameters": {
    "portal_url": "https://invoices.acme.com",
    "date_range_start": "2026-06-01",
    "date_range_end": "2026-06-15",
    "vendor_ids": ["V001", "V003"]
  }
}

CSV Bulk Run Payload:

{
  "skillId": "skill_001",
  "parameters": {
    "isBulk": true,
    "rowCount": 12,
    "csvRows": [
      {
        "portal_url": "https://invoices.company-a.com",
        "date_range_start": "2026-06-01",
        "date_range_end": "2026-06-15",
        "vendor_ids": ["V001", "V002"]
      },
      {
        "portal_url": "https://portal.invoice-hub.net",
        "date_range_start": "2026-05-01",
        "date_range_end": "2026-05-31",
        "vendor_ids": ["V009"]
      }
    ],
    // Fallback: The properties from the first CSV row are ALSO merged into the root of `parameters`
    "portal_url": "https://invoices.company-a.com",
    "date_range_start": "2026-06-01",
    "date_range_end": "2026-06-15",
    "vendor_ids": ["V001", "V002"]
  }
}
Backward Compatibility Fallback
If the backend does not support bulk CSV operations yet, the first element of csvRows is mirrored flat at the root of the parameters object so the runner can fall back to executing a single run.

Response:

{
  "success": true,
  "data": {
    "executionId": "exec_abc123",
    "status": "queued",
    "skillId": "skill_001",
    "submittedAt": "2026-06-15T10:30:00Z"
  }
}

5. Get Execution Status

GET/api/executions/:id

Monitors execution progress percentages, active steps, log traces, and final outcomes. This endpoint is polled repeatedly by the frontend client every 1.5s.

Response (Running state):

  "data": {

    "status": "running",
    "skillId": "skill_001",
    "skillName": "Extract Invoice Data",
    "submittedAt": "2026-06-15T10:30:00Z",
    "startedAt": "2026-06-15T10:30:02Z",
    "completedAt": null,
    "progress": {
      "currentStep": 3,
      "totalSteps": 8,
      "currentStepName": "Navigating to invoice list",

6. Control Actions (Pause, Resume, Cancel)

Control API calls alter execution state. To avoid UI delay, each control action must return the fully updated execution state object (matching the structure of GET /api/executions/:id).

A. Pause Execution

POST/api/executions/:id/pause

Instructs the browser runner to freeze current action steps. Status shifts to "paused".

B. Resume Execution

POST/api/executions/:id/resume

Instructs a paused runner to continue step execution. Status shifts to "running".

C. Cancel Execution

POST/api/executions/:id/cancel

Kills browser session runner. Status shifts to "cancelled".

7. Human-In-The-Loop (HITL) Actions

When the automation encounters a validation barrier (e.g. captcha validation, final transaction confirm), the status changes to "waiting_for_user". Polling stops and the user is prompted to Approve or Reject.

A. Approve HITL Step

POST/api/executions/:id/approve

Confirms proceed action. Shakes off wait locks and transitions back to "running" status.

B. Reject HITL Step

POST/api/executions/:id/reject

Cancels proceed action. Shifts execution status directly to "failed" status.

Execution Status State Table

The operational state machine has these defined statuses:

Status Name Automation Engine State Client Polling Activity
queued Awaiting slot allocation. Active (1.5s interval)
running Executing Playwright steps. Active (1.5s interval)
paused Suspended temporarily by user. Suspended (Halted)
waiting_for_user Halted awaiting manual verification. Suspended (Halted)
completed Finished operations successfully. Halted (Terminal)
failed Aborted due to error/rejection. Halted (Terminal)
cancelled Terminated by user click. Halted (Terminal)

Error Codes

Error Code HTTP Code Description / Cause
SKILL_NOT_FOUND 404 The skill identifier in request path or body does not exist.
EXECUTION_NOT_FOUND 404 The requested execution tracker ID does not exist in logs.
INVALID_PARAMETERS 400 Input values failed regex validations or data types mismatch.
MISSING_REQUIRED_PARAM 400 Form omitted a parameter marked as required: true.
EXECUTION_ALREADY_COMPLETE 409 Rejected POST modification on an execution that has ended.
SERVER_ERROR 500 Internal exception thrown by automation browser controller.
