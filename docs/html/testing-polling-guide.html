Short-Polling API Testing Guide

Status: Approved for Implementation
Method: REST Polling (Fallback)

Overview

Until the backend incorporates low-latency real-time WebSockets, the frontend tracks browser executions using REST Short-Polling. The dashboard triggers repeating HTTP GET operations on the execution record to pull step-by-step progress and operational logs.

This document guides backend developers on how to mock, test, and validate their endpoint state machinery against the frontend client's expected polling behavior.

1. Frontend Polling Lifecycle

The client uses React hooks to poll the GET /api/executions/:id endpoint at a fixed interval of 1.5 seconds (1500ms).

Status Received Polling Loop Behavior Frontend UI State Rendering Next Action Trigger
queued Active (Checks every 1.5s) Shows a waiting loader; awaiting backend initialization. Transitions to running automatically.
running Active (Checks every 1.5s) Renders progress bar fill, updates active step description, scrolls log stream. Transitions to completed, paused, waiting_for_user, or failed.
paused Suspended (Halted) Freezes progress indicators, shows warning state, alters toolbar to "Resume" button. User clicks **Resume** ➔ Sends POST .../resume, updates state, restarts polling loop.
waiting_for_user Suspended (Halted) Halts progress indicators, displays Human-in-the-loop Approve/Reject confirmation card. User clicks **Approve** (sends POST .../approve) or **Reject** (sends POST .../reject).
completed Terminated (Halted) Shows terminal success state, output summary details, and duration indicators. None (Terminal state).
failed Terminated (Halted) Shows failure red card containing the terminal error message. None (Terminal state).
cancelled Terminated (Halted) Shows cancellation indicator badge. None (Terminal state).
Why Polling Stops During Pause/HITL

Suspending the short-poll interval prevents the frontend from sending unnecessary HTTP requests to the backend while the automation runner is suspended. Polling resumes when the user clicks an action button, as the action's response returns the updated state payload.

3. Testing Backend Endpoints with cURL

Verify that your endpoints match the exact expected state properties using these standard command-line requests:

A. Trigger a New Skill Run

curl -X POST http://localhost:8080/api/executions \
  -H "Content-Type: application/json" \
  -d '{"skillId": "skill_001", "parameters": {"portal_url": "https://example.com", "date_range_start": "2026-06-01", "date_range_end": "2026-06-15", "vendor_ids": ["V01", "V02"]}}'

B. Query Polling Status (Repeat during execution)

curl -X GET http://localhost:8080/api/executions/exec_abc123

C. Force Pause Operations

curl -X POST http://localhost:8080/api/executions/exec_abc123/pause

D. Resume Halted Automation

curl -X POST http://localhost:8080/api/executions/exec_abc123/resume

E. Approve Pending Human-in-the-Loop Dialog

curl -X POST http://localhost:8080/api/executions/exec_abc123/approve

F. Reject/Abort execution

curl -X POST http://localhost:8080/api/executions/exec_abc123/reject

4. Runnable Node.js Simulation Harness

You can run this client script locally using Node.js to verify your backend's implementation of the API contract. Save as test-polling-harness.js and run node test-polling-harness.js:

/**
 * CurationPilot — Polling & Action Test Harness
 * Simulates a frontend client running against your backend API.
 */

const BASE_URL = 'http://localhost:8080/api';
const POLL_INTERVAL = 1500; // 1.5s
let activePollTimer = null;
let executionId = null;

async function callApi(path, options = {}) {
  const url = `${BASE_URL}${path}`;
  const response = await fetch(url, {
    headers: { 'Content-Type': 'application/json', ...options.headers },
    ...options
  });
  if (!response.ok) {
    const text = await response.text();
    throw new Error(`HTTP ${response.status}: ${text}`);
  }
  return response.json();
}

async function startSimulation() {
  console.log('🚀 Starting Client Execution Simulation...');
  
  try {
    const submitPayload = {
      skillId: 'skill_001',
      parameters: {
        portal_url: 'https://invoices.acme.com',
        date_range_start: '2026-06-01',
        date_range_end: '2026-06-15',
        vendor_ids: ['V001', 'V002']
      }
    };
    
    console.log('\n[1/5] Submitting execution request...');
    const result = await callApi('/executions', {
      method: 'POST',
      body: JSON.stringify(submitPayload)
    });
    
    executionId = result.data.executionId;
    console.log(`✅ Submission successful. Execution ID: "${executionId}"`);
    
    // Start Polling Loop
    runPollLoop();
    
    // Simulating user pausing after 3.5 seconds
    setTimeout(async () => {
      console.log(`\n[3/5] Simulating user click on [Pause]...`);
      const pauseResult = await callApi(`/executions/${executionId}/pause`, { method: 'POST' });
      if (pauseResult.success) {
        console.log(`⏸️ Pause acknowledged. Status returned: "${pauseResult.data.status}"`);
      }
      
      // Simulating user resuming 3 seconds later
      setTimeout(async () => {
        console.log(`\n[4/5] Simulating user click on [Resume]...`);
        const resumeResult = await callApi(`/executions/${executionId}/resume`, { method: 'POST' });
        if (resumeResult.success) {
          console.log(`▶️ Resume acknowledged. Status returned: "${resumeResult.data.status}"`);
          runPollLoop(); // Resume client-side polling loop
        }
      }, 3000);
      
    }, 3500);

  } catch (err) {
    console.error('❌ Simulation aborted:', err.message);
  }
}

function runPollLoop() {
  if (activePollTimer) clearTimeout(activePollTimer);
  
  activePollTimer = setTimeout(async () => {
    try {
      const response = await callApi(`/executions/${executionId}`);
      const { status, progress, logs } = response.data;
      
      console.log(`[POLL TICK] Status: "${status}" | Progress: ${progress?.percentage}% | Logs: ${logs?.length || 0}`);
      
      if (status === 'completed' || status === 'failed' || status === 'cancelled') {
        console.log(`\n🏁 Terminal status reached: "${status}". Halted polling.`);
        return;
      }
      
      if (status === 'paused') {
        console.log('⏸️ Poll loop suspended while execution is paused.');
        return;
      }
      
      if (status === 'waiting_for_user') {
        console.log('\n⚠️ Server waiting for HITL approval. Poll loop suspended.');
        
        // Approve after 2 seconds
        setTimeout(async () => {
          console.log('\n[HITL ACTION] Simulating user Approve click...');
          const approveResult = await callApi(`/executions/${executionId}/approve`, { method: 'POST' });
          if (approveResult.success) {
            console.log(`✅ Approved. Status returned: "${approveResult.data.status}"`);
            runPollLoop(); // Resume polling
          }
        }, 2000);
        return;
      }
      
      runPollLoop();
    } catch (err) {
      console.error('❌ Poll error:', err.message);
      runPollLoop();
    }
  }, POLL_INTERVAL);
}

startSimulation();
